# 📊 Trader Performance vs Market Sentiment

Analyzing how Bitcoin Fear & Greed sentiment relates to trader behavior and performance on Hyperliquid.

---

## 📁 Project Structure

```
├── cleaning.ipynb   ← Step 1: Data cleaning & feature engineering
├── EDA.ipynb            ← Step 2: Full analysis, charts, insights, strategy
├── historical_data.csv        ← Raw input: Hyperliquid historical trade data
├── final_dataset.csv          ← Generated by cleaning notebook (do not edit)
└── README.md
```

---

## 🚀 How to Run

**Run in order:**

1. Place `fear_greed_index.csv` and `historical_data.csv` in the same folder as the notebooks.

2. Run the cleaning notebook first:
   ```
   cleaning.ipynb
   ```
   → This generates `final_dataset.csv`

3. Run the EDA notebook:
   ```
   EDA.ipynb
   ```
   → This contains the full analysis, all charts, and strategy recommendations

> ⚠️ Do not skip Step 2. `EDA.ipynb` requires `final_dataset.csv` to exist first.

## 🔧 Feature Engineering (cleaning_corrected.ipynb)

The cleaning notebook engineers these metrics at **daily level per account**:

| Metric | Description |
|--------|-------------|
| `daily_pnl` | Total closed PnL per account per day |
| `trade_count` | Number of trades per account per day |
| `avg_trade_size` | Average position size in USD (leverage proxy) |
| `win_rate` | % of trades with positive PnL |
| `buy_sell_ratio` | Buy trades ÷ Sell trades (long/short bias proxy) |
| `sentiment_value` | Fear & Greed score for that day (0–100) |
| `sentiment_class` | Regime: Extreme Fear / Fear / Neutral / Greed / Extreme Greed |

## 📊 Analysis Summary (EDA.ipynb)

### Part A — Data Preparation
- Dataset documentation: shape, missing values, duplicates
- Timestamp alignment (daily level merge)
- Key metrics overview

### Part B — Analysis
-  Performance by sentiment: PnL, win rate, drawdown proxy
-  Behavioral changes: trade frequency, position sizing, long/short bias
-  Trader segmentation: High vs Low Frequency, Winners vs Losers
-  Account × Sentiment heatmap (primary diagnostic)
-  Scatter analysis with OLS trendlines
-  Sentiment & PnL over time
-  Feature correlation matrix

### Part C — Strategy Recommendations
- **Rule 1:** Fear-Regime Aggression — increase size and frequency during Fear/Extreme Fear
- **Rule 2:** Greed-Regime Caution — reduce leverage and tighten stops during Greed/Extreme Greed
